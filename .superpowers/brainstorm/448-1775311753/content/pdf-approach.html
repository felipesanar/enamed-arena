Como gerar o PDF da prova?

Essa escolha afeta layout, velocidade de download e complexidade técnica

A

Client-side (no browser do aluno)

Gerado em tempo real com @react-pdf/renderer. Sem custo de servidor, funciona sempre. Layout mais limitado — fontes e imagens precisam de CORS.

Prós

  • Zero custo por geração
  • Funciona offline
  • Sem latência de servidor

Contras

  • Layout menos sofisticado
  • Imagens dependem de CORS
  • Bundle maior
B

Server-side (Edge Function)

Edge Function no Supabase gera o PDF sob demanda por usuário. Controle total de tipografia, imagens e layout. Cada download gera um novo PDF.

Prós

  • Layout perfeito com qualquer fonte
  • Imagens sem problemas de CORS
  • Pode incluir nome do aluno

Contras

  • Custo por requisição
  • Latência de 2–5s por geração
  • Depende de servidor online
C

Híbrido — PDF pré-gerado por simulado

Edge Function gera o PDF uma vez por simulado (não por aluno) e armazena no Supabase Storage. Download instantâneo para todos. O nome do aluno vai no gabarito digital, não no PDF.

Prós

  • Download instantâneo
  • Layout perfeito
  • Gerado só uma vez

Contras

  • PDF sem nome do aluno
  • Precisa regenerar se questão mudar

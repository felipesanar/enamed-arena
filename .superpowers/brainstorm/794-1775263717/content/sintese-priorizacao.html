Princípios UX que devem guiar as melhorias
🔍
Discoverability First
Uma funcionalidade que não é descoberta não existe. Eliminar alternativa, alta confiança e atalhos precisam se tornar visíveis — a interface deve ensinar enquanto o usuário usa.
🧠
Fluxo adaptativo ao contexto
Novato e veterano têm necessidades diferentes no pré-prova. O produto deve reconhecer o histórico e adaptar a fricção — mais orientação para quem está começando, caminho rápido para quem já sabe.
❤️
Reforço emocional positivo
Completar uma prova de residência médica é estressante e significativo. A tela de conclusão deve celebrar o esforço — cria memória emocional positiva e aumenta a chance de retorno.
⚡
Menos cliques, mais foco
Cada clique antes da prova é ansiedade adicionada. O fluxo pré-prova para usuários recorrentes deve ser reduzido ao mínimo — o usuário veio para fazer a prova, não para confirmar que sabe as regras.
📱
Mobile como cidadão de primeira classe
O navegador de questões em sheet modal não é equivalente ao sidebar desktop. Mobile precisa de um padrão próprio que mantenha a consciência de progresso sem ocupar espaço da questão.
🎯
CTAs honestos com o estado real
"Ver resultado" antes do resultado estar disponível quebra a confiança. Cada CTA deve refletir o que o sistema pode entregar agora — não o que vai poder entregar depois.

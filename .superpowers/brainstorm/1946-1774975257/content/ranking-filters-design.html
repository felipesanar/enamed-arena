Design — Filtros do Ranking

Duas mudanças: (3a) visibilidade do filtro de segmento por tipo de usuário; (3b) instituição como sub-filtro de especialidade.

3a — Segmento: visibilidade por tipo de usuário

Cada usuário só vê os segmentos que fazem sentido para ele. Não há estado "bloqueado" — o botão simplesmente não aparece.

guest — Visitante

Segmento

Apenas "Todos" visível

standard — Aluno SanarFlix

Segmento

"Todos" + "Aluno SanarFlix"

pro — Aluno PRO

Segmento

Todos os três segmentos

Implementação: filter inline no array de opções de segmento:
segment === 'guest' → só 'all'
segment === 'standard' → 'all' + 'sanarflix'
segment === 'pro' → todos os três
Se o segmentFilter persistido não está na lista permitida → reset para 'all' com um useEffect.

3b — Instituição como sub-filtro de Especialidade

Implementação: sem mudança de tipo nem hook.
• Row principal: só botões 'all' e 'same_specialty' — botão ativo = comparisonFilter === 'same_specialty' || comparisonFilter === 'same_institution'
• Sub-filtro (só quando comparisonFilter !== 'all'): botão de instituição toggle — clique alterna entre 'same_specialty' e 'same_institution'
• Clicar em "Todos" → seta 'all' (sem mudar lógica de reset de instituição)
• ComparisonFilter type e hook ficam intactos

Ordenação — mais recente primeiro

A API já ordena por sequence_number ASC. Invertemos para DESC na query do Supabase em simuladosApi.listSimulados(). Simulado #3 aparece antes do #1 na listagem.

Onde fica cada dado — grande área

  • SimuladosPage — usa useUserPerformance() para pegar summary.last_simulado_id, carrega useSimuladoDetail + useExamResult para esse simulado, computa computePerformanceBreakdown, extrai worstArea.area
  • SimuladoCard — recebe worstArea?: string | null como nova prop. Renderiza o badge vermelho só se o valor existir
  • Cálculo acontece uma vez em SimuladosPage, passado para todos os cards

O que muda no código

  • simuladosApi.listSimulados() → .order('sequence_number', { ascending: false })
  • SimuladoCard → nova prop isLocked: boolean + worstArea?: string | null
  • SimuladosPage → computa isLocked = segment !== 'pro', passa worstArea para todos os cards
  • Card bloqueado → div clicável em vez de Link, onClick abre PRO_ENAMED_URL em nova aba
